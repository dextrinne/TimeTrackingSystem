## Описание проекта

Программная система для автоматизации процессов планирования, фиксации, согласования и контроля фактически отработанного времени сотрудниками ИММИ КубГУ.

### Назначение
- Автоматизация учёта рабочего времени
- Формирование электронного табеля
- Управление кадровыми документами (отпуска, больничные, командировки)
- Генерация отчётов и выгрузка в XLSX
- Авторизация с хешированием паролей

## Установка и запуск

### Способ 1: Запуск из исходного кода

#### 1. Требования
- Python 3.13+
- PostgreSQL 14+

#### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

#### 3. Настройка базы данных

```bash
python database/init_db.py
```

Скрипт автоматически создаст БД `time_tracking` и заполнит тестовыми данными.

#### 4. Запуск приложения
```bash
python main.py
```

### Способ 2: Запуск готового .exe

Дважды кликните по файлу `dist\TimeTrackingSystem.exe`. Консоль не открывается — приложение работает как обычная программа.

> **Важно:** для работы необходим запущенный PostgreSQL с базой данных `time_tracking`.

## Функционал системы

### 1. Справочник сотрудников (ЛР1-Ф1)
- Добавление, редактирование, удаление сотрудников
- Поиск по ФИО и должности
- Хранение ФИО, должности, ставки, нормы часов

### 2. Управление табелями (ЛР1-Ф3, ЛР2)
- Создание табеля за период (месяц)
- Автоматическое формирование структуры рабочими днями
- Ввод отработанных часов по дням
- Отметка типов дней (рабочий, отпуск, больничный и т.д.)
- Автоматический подсчёт итогов
- Управление статусами: «В работе» → «Утверждён» → «Архивирован»
- Удаление табеля с подтверждением

### 3. Управление документами (ЛР1-Ф2)
- Регистрация отпусков, больничных, командировок, отгулов
- Привязка документов к сотрудникам
- Проверка отсутствия сотрудников по датам

### 4. Формирование отчётов (ЛР1-Ф4, Ф5)
- Сводный отчёт по табелю
- Экспорт в XLSX (Excel) с форматированием
- Подсчёт итоговых показателей

### 5. Авторизация (ЛР1-НФ3, НФ4)
- Вход по логину и паролю
- Хеширование паролей (SHA-256)
- Роль: Табельщик (полный доступ ко всем функциям)

### 6. Надёжность (ЛР1-НФ1, ЛР3-Т-1)
- Создание резервной копии БД (pg_dump)
- Транзакционное выполнение запросов
- Восстановление после сбоев (rollback)

## Инструкция пользователя

### Быстрый старт

1. **Запустите приложение**: `python main.py` или `TimeTrackingSystem.exe`
2. **Войдите в систему**: логин `tabel`, пароль `tabel123`
3. **Добавьте сотрудников**: вкладка «Сотрудники» → «Добавить»
4. **Создайте табель**: вкладка «Табели» → «Создать табель»
5. **Сформируйте дни**: кнопка «🔄 Сформировать дни»
6. **Заполните часы**: двойной клик по записи
7. **Добавьте документы**: вкладка «Документы» → «Добавить документ»
8. **Сформируйте отчёт**: вкладка «Отчёты» → «Сформировать отчёт»
9. **Экспортируйте в Excel**: кнопка «Экспорт в XLSX»

### Основные операции

#### Добавление сотрудника
1. Откройте вкладку «Сотрудники»
2. Нажмите «Добавить»
3. Заполните ФИО, должность, ставку, норму часов
4. Нажмите «Сохранить»

#### Создание табеля
1. Откройте вкладку «Табели»
2. Нажмите «Создать табель»
3. Укажите период (начало и конец месяца)
4. Нажмите «🔄 Сформировать дни» для автоматического заполнения рабочими днями

#### Регистрация документа
1. Откройте вкладку «Документы»
2. Нажмите «Добавить документ»
3. Выберите сотрудника, тип документа, даты
4. Нажмите «Сохранить»

#### Формирование отчёта
1. Откройте вкладку «Отчёты»
2. Выберите табель из списка
3. Нажмите «Сформировать отчёт»
4. Для экспорта нажмите «Экспорт в XLSX»

## Инструкция администратора

### Развёртывание системы

1. Установите PostgreSQL 14+
2. Создайте базу данных: `python database/init_db.py`
3. Установите зависимости: `pip install -r requirements.txt`
4. Настройте подключение в `config.py` (если отличается от `postgres/postgres`)
5. Запустите: `python main.py`

### Сборка .exe

```bash
build.bat
```

Результат: `dist\TimeTrackingSystem.exe` (~27 МБ, без консоли).

### Резервное копирование
- Через меню: **Файл** → **Создать резервную копию БД**
- Или через pgAdmin4: Backup в формате SQL
